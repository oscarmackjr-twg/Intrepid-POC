  <action>
Create `backend/scripts/regression_test.py`. Python 3.10+. Use only stdlib (subprocess, pathlib, filecmp, os, sys, shutil, datetime, argparse). No third-party imports.

Script structure:

```
usage: python backend/scripts/regression_test.py [--test-data DIR] [--backend-dir DIR] [--pdate YYYY-MM-DD] [--tday YYYY-MM-DD]

defaults:
  --test-data: C:\Users\omack\Downloads\TestData
  --backend-dir: <repo_root>/backend  (detected relative to script location)
```

**Discovery logic:**
- Scan `--test-data` directory for subdirectories (each is a buy-date test case)
- Sort subdirectories alphabetically
- A valid test case dir must contain at least one file AND have either an `outputs/` or `output_share/` subdirectory
- Skip subdirs that don't meet this criteria with a warning

**Per-test-case execution:**
1. Determine `pdate` and `tday` for the test case:
   - If `--pdate` / `--tday` CLI args provided, use them for all cases
   - Otherwise, derive from folder name if it looks like YYYY-MM-DD; fall back to folder name as-is for `--pdate` with today's date as `--tday`
   - Print the pdate/tday being used for each case
2. Run CLI: `subprocess.run([sys.executable, "scripts/run_pipeline_cli.py", "--folder", str(test_case_dir), "--pdate", pdate, "--tday", tday], cwd=str(backend_dir), capture_output=True, text=True, timeout=300)`
3. CLI exit code 0 = pipeline ran (not necessarily outputs match); non-zero = pipeline_failed
4. Find generated outputs: scan `backend_dir/cli_debug/` for the most recently modified subdirectory created after the subprocess started — that's this run's output dir
5. If no output dir found, mark as FAILED (no outputs generated)

**Diff logic:**
For each test case that produced outputs, compare:
- `{generated_output_dir}/` vs `{test_case_dir}/outputs/` — all files recursively
- `{generated_output_dir}_share/` vs `{test_case_dir}/output_share/` — if output_share dir exists

Use `filecmp.cmp(generated_file, expected_file, shallow=False)` for byte comparison.

Report mismatches as: `  DIFF: outputs/filename.xlsx (generated differs from expected)`
Report missing files as: `  MISSING: outputs/filename.xlsx (expected but not generated)`
Report extra files as: `  EXTRA: outputs/filename.xlsx (generated but not expected)`

**Result tracking:**
```python
results = []  # list of dicts: {name, status, pipeline_ok, diffs, missing, extra, error}
```

**Report format** (print to stdout):
```
============================================================
REGRESSION TEST REPORT — {datetime}
Test data: {test_data_dir}
============================================================

[BUY DATE: {folder_name}]
  Pipeline: OK / FAILED
  Diff result: PASS (0 diffs) / FAIL ({n} differences)
  {diff/missing/extra lines if any}

... (repeat per test case)

============================================================
SUMMARY: {passed} PASSED / {failed} FAILED / {total} TOTAL
============================================================
```

**Cleanup:** After each test case, delete the generated `cli_debug/{run_id}/` directory (use `shutil.rmtree`) to keep the backend clean. Add a `--no-cleanup` flag to skip deletion for debugging.

**Error handling:** Wrap each test case in try/except — a crash in one case should not stop other cases. Print the exception and mark as FAILED.

**Exit code:** `sys.exit(0)` if all passed, `sys.exit(1)` if any failed.

Include a docstring at the top explaining usage and TestData folder structure.
