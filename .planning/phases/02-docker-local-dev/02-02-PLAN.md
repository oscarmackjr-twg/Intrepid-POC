  <action>
In `frontend/vite.config.ts`, change the proxy target from the hardcoded string to an env-var-based expression:

Before:
```typescript
target: 'http://localhost:8000',
```

After:
```typescript
target: process.env.VITE_API_TARGET ?? 'http://localhost:8000',
```

This is the only change. Preserve all other content exactly — plugins, port, changeOrigin — unchanged.

How it works:
- When running `npm run dev` on the host (no Docker), `VITE_API_TARGET` is unset → fallback `http://localhost:8000` — identical behavior to current
- When running inside the frontend Docker container, the compose env sets `VITE_API_TARGET: http://app:8000` → proxy routes to the backend container by Docker service name

No TypeScript type issues: `process.env.VITE_API_TARGET` is `string | undefined`, and `??` coalesces undefined to the fallback string. Vite's proxy `target` accepts a string.
