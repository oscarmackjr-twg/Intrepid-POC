<task type="auto">
  <name>Task 3: Add frontend service to docker-compose.yml</name>
  <files>deploy/docker-compose.yml</files>
  <action>
Add a `frontend` service to the services block. This service uses the node:20-slim image directly (NOT the production Dockerfile — that produces a static build, not a dev server):

```yaml
frontend:
  image: node:20-slim
  working_dir: /app/frontend
  volumes:
    - ../frontend:/app/frontend         # source code; HMR watches this
    - /app/frontend/node_modules        # anonymous volume preserves Linux-built binaries
  ports:
    - "5173:5173"
  environment:
    VITE_API_TARGET: http://app:8000
  command: sh -c "npm ci && npm run dev -- --host"
  depends_on:
    - app
```

Notes:
- `../frontend` is relative to `deploy/`, resolves to repo root's `frontend/`
- `- /app/frontend/node_modules` is an anonymous volume that shadows the host node_modules (Windows binaries won't overwrite Linux binaries installed by npm ci)
- `npm ci && npm run dev -- --host` installs packages fresh on each container start, then starts Vite bound to 0.0.0.0 (required for container-to-host port forwarding)
- `VITE_API_TARGET: http://app:8000` is consumed by vite.config.ts (updated in Plan 02) to proxy /api to the backend container
- No healthcheck on frontend — no other service depends on it; keeping compose simple
- `depends_on: - app` ensures the backend is at least started (not necessarily healthy) before frontend starts; avoids proxy connection errors on first request
