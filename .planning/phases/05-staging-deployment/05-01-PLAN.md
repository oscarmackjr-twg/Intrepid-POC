Update `deploy/Dockerfile` — in the frontend build stage (`FROM node:20-slim AS frontend`), add two lines immediately before `RUN npm run build`. The current file has three ENV lines before the build:

```
ENV VITE_API_URL=
ENV NODE_ENV=production
ENV NODE_OPTIONS=--max-old-space-size=4096
```

Add after those three lines, before `RUN npm run build`:

```dockerfile
ARG VITE_APP_ENV
ENV VITE_APP_ENV=$VITE_APP_ENV
```

The `ARG` declaration must be in the `frontend` stage (before `RUN npm run build`) not in the backend stage. Without `ARG VITE_APP_ENV`, the `--build-arg` is silently ignored by Docker.

Update `.github/workflows/deploy-test.yml` — in the "Build and push image" step, add `--build-arg VITE_APP_ENV=staging \` to the `docker build` command. The updated command should be:
