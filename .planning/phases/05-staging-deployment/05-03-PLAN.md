<task type="auto">
  <name>Task 2: Run staging seed script via ECS one-off task</name>
  <files></files>
  <action>
Run the admin seed script as an ECS one-off task (same pattern as Alembic migrations in the deploy workflow). Use PowerShell syntax.

```powershell
# Get subnet and security group from Terraform outputs
cd C:\Users\omack\Intrepid\pythonFramework\intrepid-poc\deploy\terraform\qa
$SUBNET_IDS = terraform output -raw ecs_subnet_ids
$SG_ID = terraform output -raw ecs_security_group_id
echo "Subnets: $SUBNET_IDS"
echo "SG: $SG_ID"

# Run seed script
$TASK_ARN = aws ecs run-task `
  --cluster intrepid-poc-qa `
  --task-definition intrepid-poc-qa `
  --launch-type FARGATE `
  --network-configuration "awsvpcConfiguration={subnets=[$SUBNET_IDS],securityGroups=[$SG_ID],assignPublicIp=ENABLED}" `
  --overrides '{\"containerOverrides\":[{\"name\":\"app\",\"command\":[\"python\",\"scripts/seed_staging_user.py\"]}]}' `
  --query 'tasks[0].taskArn' `
  --output text
echo "Seed task started: $TASK_ARN"

# Wait for completion (up to ~5 min)
aws ecs wait tasks-stopped --cluster intrepid-poc-qa --tasks $TASK_ARN

# Check exit code — must be 0
$EXIT_CODE = aws ecs describe-tasks `
  --cluster intrepid-poc-qa `
  --tasks $TASK_ARN `
  --query 'tasks[0].containers[?name==`app`].exitCode' `
  --output text
echo "Exit code: $EXIT_CODE"
```

If exit code is non-zero, check CloudWatch Logs:
```powershell
# Get the log stream name for the seed task
aws logs get-log-events `
  --log-group-name /ecs/intrepid-poc-qa `
  --log-stream-name "ecs/app/$($TASK_ARN.Split('/')[-1])"
```

Common failure: `ModuleNotFoundError: No module named 'auth'` — this means the command path is wrong. The command must be `["python","scripts/seed_staging_user.py"]` (relative, from /app WORKDIR). Verify the override is using this exact format.
  </action>
  <verify>
    <automated>aws ecs describe-tasks --cluster intrepid-poc-qa --tasks $TASK_ARN --query 'tasks[0].containers[?name==`app`].exitCode' --output text</automated>
  </verify>
  <done>
    - Seed task completed with exit code 0
    - CloudWatch Logs show "Admin user created: admin" or "Admin user updated: admin"
  </done>
</task>
