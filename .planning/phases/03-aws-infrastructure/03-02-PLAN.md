<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: RDS connectivity verification (INFRA-04)</name>
  <what-built>
    Secrets Manager entries verified (INFRA-02) and ECR push test completed (INFRA-03). The final verification is RDS reachability (INFRA-04), which requires a temporary security group ingress rule from the local machine's IP before psql can connect.
  </what-built>
  <how-to-verify>
    Run these steps in order. All commands from repo root (bash).

    **Step 1 — Get the RDS security group ID:**
    ```bash
    RDS_SG_ID=$(aws ec2 describe-security-groups \
      --filters "Name=group-name,Values=intrepid-poc-qa-rds-sg-*" \
      --query 'SecurityGroups[0].GroupId' \
      --output text \
      --region us-east-1)
    echo "RDS SG: $RDS_SG_ID"
    ```

    **Step 2 — Get your current public IP:**
    ```bash
    MY_IP=$(curl -s https://checkip.amazonaws.com)
    echo "Your IP: $MY_IP"
    ```

    **Step 3 — Add temporary ingress rule for port 5432 from your IP:**
    ```bash
    aws ec2 authorize-security-group-ingress \
      --group-id "$RDS_SG_ID" \
      --protocol tcp \
      --port 5432 \
      --cidr "${MY_IP}/32" \
      --region us-east-1
    ```

    **Step 4 — Get the RDS endpoint:**
    ```bash
    RDS_ENDPOINT=$(terraform -chdir=deploy/terraform/qa output -raw rds_endpoint)
    echo "RDS Endpoint: $RDS_ENDPOINT"
    ```

    **Step 5 — Connect via psql:**
    ```bash
    psql "postgresql://postgres:Intrepid456\$%@${RDS_ENDPOINT}:5432/intrepid_poc?sslmode=require" \
      -c "SELECT version();"
    ```
    Expected: Returns PostgreSQL version string. Example: `PostgreSQL 16.8 on x86_64-pc-linux-gnu...`

    **IMPORTANT — Step 6 — Revoke the temporary ingress rule:**
    ```bash
    aws ec2 revoke-security-group-ingress \
      --group-id "$RDS_SG_ID" \
      --protocol tcp \
      --port 5432 \
      --cidr "${MY_IP}/32" \
      --region us-east-1
    ```
    Do not skip this step — leaving public 5432 ingress open is a security risk.

    **Troubleshooting:** If psql times out (not refused), the ingress rule did not apply yet — wait 30 seconds and retry. If SSL error, try removing `?sslmode=require` (RDS may use `sslmode=prefer` negotiation).
  </how-to-verify>
  <resume-signal>Type "verified" when psql returns the PostgreSQL version string and the ingress rule has been revoked. Or describe the error encountered.</resume-signal>
