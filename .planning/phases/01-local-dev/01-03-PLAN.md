CRITICAL: Makefile recipes MUST use tab indentation (not spaces). If the Write tool produces spaces, the make command will fail with "missing separator" error. Verify indentation after writing.

```makefile
.PHONY: setup run-backend run-frontend migrate

# Install backend Python dependencies into venv and frontend npm packages.
# Run once after cloning, or when requirements.txt / package.json change.
setup:
	cd backend && python -m venv venv
	cd backend && venv/bin/pip install -r requirements.txt
	cd frontend && npm install

# Start the FastAPI backend with auto-reload at http://localhost:8000
# Requires backend/.env to exist (copy from backend/.env.example and fill in DATABASE_URL).
run-backend:
	cd backend && venv/bin/uvicorn api.main:app --reload --host 0.0.0.0 --port 8000

# Start the Vite frontend dev server with hot reload at http://localhost:5173
# The Vite proxy forwards /api requests to localhost:8000 — run-backend must be running.
run-frontend:
	cd frontend && npm run dev

# Apply Alembic database migrations to local Postgres.
# Requires DATABASE_URL in backend/.env and the intrepid_poc database to exist.
# If you have a pre-Alembic database, drop and recreate it first:
#   psql -U postgres -c "DROP DATABASE IF EXISTS intrepid_poc; CREATE DATABASE intrepid_poc;"
migrate:
	cd backend && venv/bin/alembic upgrade head
```

Notes on discretion:
- Use venv/bin/ (Unix path) as the primary; Windows developers may need to adjust to venv/Scripts/ — document this in DEVELOPMENT.md
- Port 8000 for backend, 5173 for frontend (matches vite.config.ts)
- --host 0.0.0.0 allows access from Docker in Phase 2 without Makefile changes
- No `make test` target (deferred per user decision; can be added later without breaking changes)

Create DEVELOPMENT.md at the project root. This document must be complete enough that a developer can get from a fresh clone to a running local stack using only this file and backend/.env.example. No undocumented steps.

Content to cover (in order):

**1. Prerequisites**
- Python 3.11+ (check: `python --version`)
- Node.js 18+ and npm (check: `node --version`)
- PostgreSQL running locally (check: `psql -U postgres -c '\l'`)
  - Database to create: `psql -U postgres -c "CREATE DATABASE intrepid_poc;"`
- Git

**2. Initial Setup**
```bash
git clone <repo-url>
cd intrepid-poc
make setup
```

Explain what `make setup` does: creates `backend/venv`, installs Python deps, runs `npm install` for frontend.

Note for Windows: `make` requires GNU Make. Alternative commands if make is unavailable:
```
cd backend && python -m venv venv && venv\Scripts\pip install -r requirements.txt
cd frontend && npm install
```

**3. Environment Configuration**
```bash
cp backend/.env.example backend/.env
```
Edit `backend/.env`:
- `DATABASE_URL` — update if your Postgres credentials differ from the default (`postgres:postgres`)
- `SECRET_KEY` — change to any random string for local dev
- Leave `STORAGE_TYPE=local` and `DEV_INPUT=./data/sample/files_required` as-is for sample data

**4. Run Database Migrations**
```bash
make migrate
```
If you have an existing `intrepid_poc` database from before Alembic was introduced, drop and recreate it:
```bash
psql -U postgres -c "DROP DATABASE IF EXISTS intrepid_poc; CREATE DATABASE intrepid_poc;"
make migrate
```

**5. Start the Backend**
```bash
make run-backend
```
Backend starts at `http://localhost:8000`. Verify: `curl http://localhost:8000/api/health`

Note: Always start the backend before the frontend — the Vite dev server proxies `/api` requests to `:8000`.

**6. Start the Frontend**
```bash
make run-frontend
```
Frontend starts at `http://localhost:5173` with hot reload.

**7. Running the Pipeline Locally (LOCAL-06)**

The app includes a sample dataset at `backend/data/sample/files_required/` committed to the repository. The pipeline's input file discovery uses date-stamped filenames. Sample files are dated `02-18-2026` (meaning the pipeline processing date is `2026-02-19`).

Via the UI: Use the run form and enter processing date `2026-02-19`.

Via CLI directly:
```bash
cd backend
venv/bin/python main.py --pdate 2026-02-19
```

To use your own live input files instead of sample data, set `DEV_INPUT` in `backend/.env` to point at your `files_required/` directory:
```
DEV_INPUT=/absolute/path/to/your/files_required
```

**8. Development Notes**
- All backend commands must be run from the `backend/` directory (pydantic-settings resolves `.env` relative to cwd)
- The Makefile handles the `cd backend` step automatically
- Alembic enum changes (UserRole, RunStatus) are not auto-detected — write manual migration steps if you modify these enums
- `backend/.env` is gitignored — never commit it. Only `backend/.env.example` is tracked.

**9. Troubleshooting**
- `ModuleNotFoundError: No module named 'api'` — you ran uvicorn from the wrong directory. Run `make run-backend` or `cd backend && uvicorn api.main:app`
- `connection refused` on DATABASE_URL — verify Postgres is running and `intrepid_poc` database exists
- Pipeline `FileNotFoundError: Tape20Loans file not found` — running without `--pdate` or wrong date. Use `--pdate 2026-02-19` with sample data.
- `STORAGE_TYPE=s3` errors without S3 credentials — `backend/.env` still has `STORAGE_TYPE=s3`. Set to `STORAGE_TYPE=local`.
