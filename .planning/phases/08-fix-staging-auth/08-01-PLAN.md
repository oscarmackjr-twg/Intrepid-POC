<objective>
Apply the LOCAL_DEV_MODE=true ECS environment variable to AWS by running terraform plan + apply, and add the same variable to docker-compose.yml for local parity.

Purpose: The root cause of staging authentication failure (MISS-02) is that the ECS task runs with LOCAL_DEV_MODE=false (default), causing FastAPI to set `secure=True` on the login cookie. HTTP ALB silently drops secure cookies, so login appears to succeed but the session cookie is never stored — all subsequent requests return 401. The fix (LOCAL_DEV_MODE=true in ecs.tf) was written in Phase 7 but terraform apply was never run. This plan executes that apply and closes the local parity gap (MISS-01) in docker-compose.yml.

Output:
- deploy/docker-compose.yml with LOCAL_DEV_MODE: "true" added to app environment block
- New ECS task definition revision with LOCAL_DEV_MODE=true active in AWS
- Terraform state synced to current AWS reality
</objective>
